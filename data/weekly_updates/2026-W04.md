# 2026-W04（2026-01-12～2026-01-18）

## Weekly Summary

本周的核心是 skills、memory、CLI/context management 和浏览器/搜索工具进入 agent 产品底层。Codex、Cursor、Copilot、OpenClaw、TRAE 都在不同层面把“提示词”升级为可管理能力。

## Head Product Signals

- **OpenAI Codex**：1 月 14 日 GPT-5.2-Codex API availability，说明 Codex 模型能力开始对外成为可集成资源。
- **Claude Code**：2.1.6-2.1.12 连续迭代，偏 CLI 稳定性与工程配置；信号不如产品大版本强，但 release 节奏值得关注。
- **Cursor**：1 月 16 日 CLI Agent Modes and Cloud Handoff，把本地终端任务与云端延续连接起来。
- **GitHub Copilot**：1 月 14-16 日 Copilot SDK technical preview、CLI enhanced agents/context management、agentic memory、BYOK/OpenCode 支持集中出现。
- **Windsurf / Devin Desktop**：未观察到同等级大版本；Devin 的云端执行模型仍作为 later-stage autonomous agent 对照。
- **OpenClaw**：1 月 12-16 日 channels 重命名、memory vector search、web_search/web_fetch、browser control、hooks、inbound media understanding 同时推进。
- **Kimi Code**：回溯补录：本周未发现 Kimi Code 同级重大公开更新；先把 Kimi 模型进入 coding 场景作为 2026 基线观察，重点看模型能力是否会被产品化为 CLI、IDE、ACP、goal workflow。
- **Zhipu GLM Coding Plan / CodeGeeX**：回溯补录：本周未发现智谱/CodeGeeX 同级重大公开更新；以 CodeGeeX IDE 插件和 GLM Coding Plan 的模型套餐路线作为国内模型厂商入口基线。
- **TRAE SOLO**：TRAE v3.3.21 出现 SOLO Skills、Browser DevTool 入口、Mermaid DocView 等信号，说明国内路线也开始把技能、浏览器和文档产物放进同一工作流。

## Competitive Reading

- **路线判断**：Skills 的本质不是 prompt 收藏夹，而是把模型能力产品化：输入模板、工具权限、验收标准、复用边界、团队治理。
- **模型能力变化**：GPT-5.2-Codex API 与 Copilot 多模型/BYOK 信号说明模型会继续快速换代，产品不能把能力绑定死在单模型上。
- **产品能力补位**：产品层要做 skills registry、memory provenance、browser/search 权限和 hook 生命周期，否则用户无法判断 agent 为什么这样做。

## TRAE SOLO Implication

TRAE SOLO 的 Skill 体系应按角色和产物组织：竞品分析、投研 memo、落地页、数据分析、代码修复都应有不同输入/验收模板。

## LPME Implication

LPME 增加“技能可复用性”评分：是否说明适用场景、输入材料、工具权限、输出格式和验证方式。
