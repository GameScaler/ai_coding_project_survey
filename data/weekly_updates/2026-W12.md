# 2026-W12（2026-03-09～2026-03-15）

## Weekly Summary

本周重点是 marketplace、fast mode、dashboard、code review CLI 和新输入方式。产品开始把 agent 从“会做事”进一步包装成“可被团队发现、配置、调用、观测”的系统。

## Head Product Signals

- **OpenAI Codex**：3 月 11-12 日 Codex app 持续更新；本周无新模型大版本。
- **Claude Code**：2.1.72-2.1.76 包括 1M context default for Opus 4.6、/context 建议、MCP elicitation；上下文与交互输入更结构化。
- **Cursor**：3 月 11 日 New Plugins on Cursor Marketplace；插件市场成为团队能力扩展入口。
- **GitHub Copilot**：3 月 11 日 GitHub CLI 请求 Copilot code review，JetBrains agentic capabilities improvements，3 月 13 日 skip approval for Actions workflows。
- **Windsurf / Devin Desktop**：3 月 11-13 日 Devin Datadog MCP、on-demand session insights、inline voice recording、@ mention sessions。
- **OpenClaw**：3 月 12 日 Control UI dashboard-v2、OpenAI/Claude fast mode；3 月 13 日 official Chrome DevTools MCP attach mode 与移动聊天设置刷新。
- **TRAE SOLO**：本周 TRAE 无同等级大版本；但 Markdown/terminal 可读性应继续和 dashboard/insights 方向合流。

## Competitive Reading

- **路线判断**：Marketplace 和 dashboard 说明 agent 产品开始变成平台。用户需要知道有哪些工具、技能、会话、成本和风险，而不是只看最终回答。
- **模型能力变化**：1M context、fast mode、skip approval 都是模型能力与产品控制的交界处：更强/更快/更自动必须配合可见度。
- **产品能力补位**：产品能力要补工具市场治理、session insights、结构化人工输入、approval skip 的风险解释。

## TRAE SOLO Implication

TRAE SOLO 可以把“技能市场”做得更业务化：不是插件列表，而是角色任务货架 + 输出样例 + 验收方式。

## LPME Implication

LPME 增加“能力发现成本”指标：用户能否快速找到正确 skill/tool/model，而不是靠猜 prompt。
