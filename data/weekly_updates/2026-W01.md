# 2026-W01（2025-12-22～2025-12-28）

## Weekly Summary

本周处在 2026 竞争周期的起点，公开大版本信号较少，但 2025 年底已经给出清晰基线：coding 产品不再只是 autocomplete，而是在往 skills、review、agent session 和团队治理延伸。

## Head Product Signals

- **OpenAI Codex**：本周无重大公开更新；12 月中旬的 GPT-5.2-Codex 与 Agent skills 仍构成进入 2026 的能力基线。
- **Claude Code**：npm 包在 2.0.76 附近小步迭代，公开信号以 CLI 稳定性为主；Claude Code 的核心心智仍是 terminal-native、可执行、可审查。
- **Cursor**：12 月 22 日 Cursor 2.3 强调 layout customization 和稳定性，说明 agent workspace 需要先把多面板、多任务的基础体验做稳。
- **GitHub Copilot**：12 月中旬的 Copilot agent skills、模型更新和 code review 信号仍在延续；GitHub 的优势是 issue/PR/review 的工作流入口。
- **Windsurf / Devin Desktop**：本周未观察到同等级公开大版本；Windsurf/Devin 仍以 IDE agent 与 autonomous engineer 两条线并行。
- **OpenClaw**：本周未观察到核心公开更新；后续应从 gateway、channels、plugins 和 agent runtime 角度跟踪，而不是按 IDE 逻辑理解它。
- **Kimi Code**：本周无同等级重大公开更新；作为 2026 基线，重点看模型能力是否会被产品化为 CLI、IDE、ACP、goal workflow。
- **Zhipu GLM Coding Plan / CodeGeeX**：本周无同等级重大公开更新；作为 2026 基线，关注 CodeGeeX IDE 插件和 GLM Coding Plan 模型套餐这条国内模型厂商入口。
- **TRAE SOLO**：本周未观察到 SOLO 大版本公开信号；对 TRAE 来说，2026 起点应先明确 More Than Coding 的产品北极星。

## Competitive Reading

- **路线判断**：竞争起点不是谁补全更快，而是谁能把模型能力包进可复用的 workflow、skills、review 和团队上下文。
- **模型能力变化**：模型侧进入 GPT-5.2 / Claude 4.x / 多模型组合时代，但产品体验仍大量依赖人工拆任务、补上下文、做验证。
- **产品能力补位**：产品层的第一优先级是把上下文、计划、执行记录和产物验收串起来；没有这个闭环，强模型也会变成高成本聊天。

## TRAE SOLO Implication

TRAE SOLO 不应把目标设成“另一个 IDE agent”，而应从第一天把非工程师的任务交付作为边界：资料、文档、表格、PPT、网页、代码都只是可交付物的不同形态。

## LPME Implication

LPME 初版需要保留“无重大更新周”的扫描能力：benchmark 不只测发布日，也要测产品在稳定期是否持续改善可靠性。
