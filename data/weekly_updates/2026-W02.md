# 2026-W02（2025-12-29～2026-01-04）

## Weekly Summary

跨年周公开更新较少，适合把头部产品按战略位置重新归类：模型公司做多 surface，IDE 公司做 agent workspace，平台公司做 workflow ownership，开源/运行时路线做可部署 agent gateway。

## Head Product Signals

- **OpenAI Codex**：无重大公开更新；进入 2026 的关键问题是 Codex 如何把模型升级转化为 CLI、IDE、App、cloud task 的一致体验。
- **Claude Code**：无重大功能信号；Claude Code 的产品护城河仍是透明执行、shell/Git/test 流程和工程师信任。
- **Cursor**：无重大公开更新；12 月底的布局和稳定性改动说明 Cursor 正在为后续 Agents Window 做体验铺垫。
- **GitHub Copilot**：无重大公开更新；GitHub 继续拥有 issue、PR、Actions、code review 的天然场景闭环。
- **Windsurf / Devin Desktop**：无重大公开更新；本地 IDE 和云端 autonomous agent 的融合仍是后续观察主线。
- **OpenClaw**：无重大公开更新；但它应被纳入主要产品池，因为它代表 agent runtime / channel gateway 方向。
- **Kimi Code**：本周无同等级重大公开更新；作为 2026 基线，重点看模型能力是否会被产品化为 CLI、IDE、ACP、goal workflow。
- **Zhipu GLM Coding Plan / CodeGeeX**：本周无同等级重大公开更新；作为 2026 基线，关注 CodeGeeX IDE 插件和 GLM Coding Plan 模型套餐这条国内模型厂商入口。
- **TRAE SOLO**：无重大公开更新；TRAE SOLO 的机会在于把国内协作入口和多角色任务放进同一交付工作台。

## Competitive Reading

- **路线判断**：这一周的判断是“先定坐标系”。AI coding 产品会分成 coding surface、workflow platform、agent runtime、business workbench 四类能力，不同产品会跨类竞争。
- **模型能力变化**：模型版本本身没有重大变化，但多模型接入已经成为默认预期。用户不会长期只用一个模型完成所有任务。
- **产品能力补位**：产品能力要负责模型选择、权限边界、上下文召回、结果验收和成本解释。

## TRAE SOLO Implication

TRAE SOLO 需要把“Work/Code 双模式”设计成同一个 agent state 的两种视图，而不是两个割裂产品。

## LPME Implication

LPME 应从一开始记录产品形态差异：IDE 内效率、终端透明度、平台工作流、跨角色产物交付要分维度评分。
