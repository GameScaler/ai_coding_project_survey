# 2026-W03（2026-01-05～2026-01-11）

## Weekly Summary

本周开始出现 2026 年第一批横向信号：Cursor 强化 CLI，GitHub 给 Copilot 多 IDE 模型入口，OpenClaw 在安全、沙箱、channels、Codex CLI fallback 与插件化上快速推进。

## Head Product Signals

- **OpenAI Codex**：本周无重大公开更新；Codex 暂时不是新闻中心，但其模型/API 基线会影响 Cursor、GitHub、OpenClaw 等下游接入。
- **Claude Code**：2.1.0 后连续小版本，说明 terminal agent 的 release cadence 已经非常高；关注点是 session、CLI、工具执行稳定性。
- **Cursor**：1 月 8 日发布 CLI features 与性能改进，说明 Cursor 不只做 IDE，也在争夺 terminal / cloud handoff 入口。
- **GitHub Copilot**：1 月 6 日 Gemini 3 Flash 进入 Visual Studio、JetBrains、Xcode、Eclipse，说明 Copilot 继续用多 IDE + 多模型覆盖企业用户。
- **Windsurf / Devin Desktop**：本周未观察到重大公开更新；Devin 仍是异步 autonomous engineer 的对照组。
- **OpenClaw**：1 月 5-11 日出现 image tool、DM 安全默认锁定、sandbox 范围、Teams channel、Codex CLI fallback、plugins first-class 等信号。
- **Kimi Code**：回溯补录：本周未发现 Kimi Code 同级重大公开更新；先把 Kimi 模型进入 coding 场景作为 2026 基线观察，重点看模型能力是否会被产品化为 CLI、IDE、ACP、goal workflow。
- **Zhipu GLM Coding Plan / CodeGeeX**：回溯补录：本周未发现智谱/CodeGeeX 同级重大公开更新；以 CodeGeeX IDE 插件和 GLM Coding Plan 的模型套餐路线作为国内模型厂商入口基线。
- **TRAE SOLO**：本周未观察到同等级大版本；可借鉴的是 OpenClaw 对安全默认值和 channel 权限的早期重视。

## Competitive Reading

- **路线判断**：产品竞争不只发生在编辑器内。CLI、channel、plugin、sandbox 会成为 agent 可部署性的基础设施。
- **模型能力变化**：模型能力开始多路供应，产品必须处理不同模型在工具调用、多模态、上下文长度上的差异。
- **产品能力补位**：产品层要把危险默认值改成安全默认值：DM 权限、沙箱范围、工具 allowlist、session 隔离都要前置。

## TRAE SOLO Implication

TRAE SOLO 面向非工程师时更需要默认安全：读取哪些资料、连接哪些账号、能不能发消息、能不能改线上内容，都要用业务语义表达。

## LPME Implication

LPME 增加“默认安全与权限可理解性”观察项，检查新手是否知道 agent 能访问什么、不能访问什么。
