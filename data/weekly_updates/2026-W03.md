# 2026-W03（2026-01-05～2026-01-11）

## Weekly Summary

本周的关键判断是：AI coding 产品的差异化正在从“接入哪个模型”转向“如何把模型放进可复用的工作系统”。Rules、Skills、MCP、memory、project instructions 这些看似配置项的能力，会变成组织级知识资产。

## Product Signals

- 公开重大更新较少，但多个头部产品都在向规则、技能、外部工具和项目上下文的方向演化。
- 用户开始需要跨会话复用偏好、代码规范、业务约束和常用操作。

## Product Interpretation

- **路线判断**：模型泛化能力越强，产品越要给它边界。Rules / Skills 的本质不是 prompt 管理，而是把组织经验固化成 agent 可调用的能力包。
- **对 TRAE SOLO 的启示**：面向 PM、运营、销售、投研等角色，Skill 不应只是一段技术 prompt，而应包含输入模板、执行步骤、验收方式、引用规范和最终产物格式。

## LPME Implication

- LPME 应增加“能否复用项目规则”的观察项：同一任务二次运行时，产品是否自动继承品牌规范、数据口径、代码风格和交付模板。
