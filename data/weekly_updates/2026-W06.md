# 2026-W06（2026-01-26～2026-02-01）

## Weekly Summary

本周的强信号来自 Codex 默认开启 web search、GitHub Agents tab / ACP support、OpenClaw rebrand 与 CLI/安全硬化。agent 正在从单会话工具变成可发现、可管理、可接入协议的系统。

## Head Product Signals

- **OpenAI Codex**：1 月 28 日 web search enabled by default，说明 Codex 把外部知识获取作为默认 agent 能力，而不是用户手动补资料。
- **Claude Code**：2.1.20-2.1.29 连续发布，仍偏工程稳定性；高频发布本身说明 CLI agent 进入快速产品化周期。
- **Cursor**：本周无重大公开更新；上周的 subagents/skills 继续构成主线。
- **GitHub Copilot**：1 月 26 日 repository Agents tab，1 月 28 日 ACP support in Copilot CLI，GitHub MCP Server 新增 Projects tools / OAuth scope filtering。
- **Windsurf / Devin Desktop**：无重大公开更新；后续需要重点观察是否把 agent fleet 管理做成主界面。
- **OpenClaw**：1 月 29 日 rebrand 为 openclaw / @openclaw/*，并补 CLI completion、per-agent model status、plugin/hook path validation。
- **Kimi Code**：本周无同等级重大公开更新；在 skills、memory、channels、cloud agent 逐步成形的阶段，Kimi 后续要重点观察“模型升级 + agent harness”是否同步推进。
- **Zhipu GLM Coding Plan / CodeGeeX**：本周无同等级重大公开更新；CodeGeeX 的观察重点从代码补全/解释扩展到 review、test、MCP 和团队工作流。
- **TRAE SOLO**：本周无同等级大版本；但应关注 web search 默认化对非工程角色的重要性，行业研究/数据分析离不开外部信息。

## Competitive Reading

- **路线判断**：默认 web search、Agents tab、ACP support 共同指向一个判断：agent 需要被产品系统发现、调度、接入，而不是藏在聊天框里。
- **模型能力变化**：模型能力外延到搜索和协议接入后，产品要负责可信来源、引用、权限和成本。
- **产品能力补位**：产品层要把外部信息获取纳入可审查链路：来源、时间、引用、是否过期、是否能复现。

## TRAE SOLO Implication

TRAE SOLO 做行业研究、投研、营销时，应把 web/doc/source citation 作为一等能力，而不是“联网搜索一下”的隐藏工具。

## LPME Implication

LPME 增加“资料来源与引用可信度”维度，测 agent 是否能把外部信息变成可审查证据。
