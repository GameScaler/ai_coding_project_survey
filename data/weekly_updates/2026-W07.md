# 2026-W07（2026-02-02～2026-02-08）

## Weekly Summary

本周是 Codex App 与 OpenClaw Feishu/channel 能力的关键周。GitHub 同时把 Claude/Codex 带进 public preview，竞争焦点从单模型能力转向“哪个产品能把多模型 agent 放进真实工作流”。

## Head Product Signals

- **OpenAI Codex**：2 月 2 日 Introducing the Codex app，2 月 5 日 Introducing GPT-5.3-Codex；Codex 从模型/CLI 向独立 App surface 扩张。
- **Claude Code**：2.1.30-2.1.37 继续快速修复；Claude Code 仍以高频 CLI 迭代保持专业用户黏性。
- **Cursor**：本周无重大公开更新；后续 2 月中旬会进入 long-running agents 与 sandbox/subagent 节奏。
- **GitHub Copilot**：2 月 4 日 Claude and Codex public preview on GitHub，2 月 5 日 Claude Opus 4.6 GA for Copilot，显示 Copilot 的模型货架策略。
- **Windsurf / Devin Desktop**：2 月 6 日 Devin release notes 包含 Schedule from input box 等自动化入口，Devin 继续强化异步任务触发。
- **OpenClaw**：2 月 2 日 Feishu/Lark plugin support、Web UI Agents dashboard、subagents guidance；这是把 agent 接进国内协作渠道的重要信号。
- **Kimi Code**：本周无同等级重大公开更新；在 skills、memory、channels、cloud agent 逐步成形的阶段，Kimi 后续要重点观察“模型升级 + agent harness”是否同步推进。
- **Zhipu GLM Coding Plan / CodeGeeX**：本周无同等级重大公开更新；CodeGeeX 的观察重点从代码补全/解释扩展到 review、test、MCP 和团队工作流。
- **TRAE SOLO**：TRAE v3.3.29 继续围绕 Skills 优化修复；当 Skills 进入真实使用，稳定性和发现机制比“能创建”更重要。

## Competitive Reading

- **路线判断**：App surface、public preview、多模型货架、Feishu channel 同时出现，说明头部产品都在争夺 agent 的分发入口。
- **模型能力变化**：GPT-5.3-Codex 和 Claude Opus 4.6 进入不同产品，模型差异会直接影响同一产品的任务边界和成本策略。
- **产品能力补位**：产品层要做模型路由、任务分配和权限解释：哪个模型适合规划、哪个适合编码、哪个适合审查。

## TRAE SOLO Implication

TRAE SOLO 的飞书/国内协作入口是战略资产，不能只把它当消息推送；应让文档、群、任务、评论成为 agent 上下文与交付链路。

## LPME Implication

LPME 加入“多模型可解释选择”指标，记录产品是否说明为何用某个模型/模式。
