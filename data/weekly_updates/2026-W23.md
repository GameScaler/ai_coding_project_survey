# 2026-W23（2026-06-01～2026-06-07）

## Weekly Summary

本周 AI coding 产品的主线是从“更强模型”转向“可托管、可审查、可恢复的 agent 系统”。Codex Sites、Cursor SDK auto-review、GitHub Copilot agent APIs/sandboxes、Claude Code fallbackModel、OpenClaw runtime/channel hardening、Kimi Code sub-skill/goal queue，以及 TRAE SOLO 浏览器元素上下文，都在补模型长任务执行时的部署、权限、上下文和恢复短板。

## Head Product Signals

- **OpenAI Codex**：6 月 1-4 日新增 Amazon Bedrock provider、Sites preview、iOS Codex 安全锁/默认 follow-up 设置/Windows SSH 连接、Profile activity insights/share cards，并继续发布 CLI 0.136/0.137；Codex 从代码生成继续扩到托管网站、跨端控制和企业云身份/计费接入。
- **Claude Code**：2.1.166-2.1.168 的核心是 fallbackModel、deny rule glob、跨 session 消息权限收紧、thinking 开关和 remote/background session 修复；产品层在补模型不可用、权限误传和长会话恢复这些 agent 稳定性问题。

## Competitive Reading

- **路线判断**：AI coding 正在分成两条产品线：一条是 IDE/CLI agent 的执行层，另一条是托管、审查、回写、计费、权限和团队治理的系统层；后者会决定模型能力能否稳定交付。
- **模型能力变化**：本周不是单一大模型版本发布，而是多产品把多模型、fallback、reasoning effort、thinking 开关、Gemini/MAI-Code 等能力做成可配置的产品开关。
- **产品能力补位**：模型仍容易在长任务中遇到上下文缺口、工具权限风险、输出污染、会话中断和部署断点；产品正在用 auto-review、sandbox、fallbackModel、Sites、custom tools、sub-skills、channel native approvals 和 runtime recovery 补这些短板。

## TRAE SOLO Implication

TRAE SOLO 本周自己的浏览器元素选取是正确方向：面向非工程师时，关键不是让用户学会更精确 prompt，而是把网页元素、评论位置、业务对象和验收点直接变成 agent 上下文。下一步应把“选择元素 -> 生成任务 -> 执行 -> 预览 -> 评论回写”做成闭环，而不是只停留在对话引用。

## LPME Implication

LPME 更新“可控长任务交付”维度：新增对可视化上下文捕获、工具/命令自动审查、模型 fallback、长任务队列、跨端恢复、部署托管、channel 回写和敏感中间推理不外泄的评分。TRAE SOLO 需要重点验证非工程师是否能在不理解 shell、MCP、模型 effort 的情况下完成同等控制。
