# 2026-W23（2026-05-25～2026-05-31）

## Weekly Summary

本周最重要的竞争主题是“模型升级正在释放更长 agent workflow，但产品必须补上控制层”。Claude Code 的 Opus 4.8 / dynamic workflows 和 Cursor 3.6 的 Auto-review Run Mode 从两侧回答同一个问题：如何让 agent 做更大的任务，同时不失控。

## Product Signals

- Claude Code 2.1.154 引入 Opus 4.8、high effort、xhigh effort、dynamic workflows、fast mode 成本优化。
- Claude Code 随后连续修复 Opus 4.8、dynamic workflow、auto mode、第三方 provider、IDE/subagent 稳定性问题。
- Cursor 3.6 发布 Auto-review Run Mode，让 Cursor 能运行更久、减少 approval prompts，并保持更安全执行。
- TRAE SOLO 在 5 月底以修复为主，没有同等级别大版本公开更新。

## Product Interpretation

- **路线判断**：模型能力提升后，产品不能只开放更大自由度。更合理的路径是同步增加 approval policy、sandbox、worktree、checkpoint、verifier、review mode。
- **对 TRAE SOLO 的启示**：面向非工程师，控制层要业务语义化。用户不应该确认“是否执行某条 shell 命令”，而应该确认“是否读取这份资料”“是否采用这个方案”“是否发布到这个空间”“是否通知协作者”。

## LPME Implication

- LPME v0.2 应新增：
  - model upgrade sensitivity；
  - long-running autonomy control；
  - approval friction；
  - rollback/checkpoint 可见性。
