# 2026-W20（2026-05-04～2026-05-10）

## Weekly Summary

本周核心信号是 TRAE SOLO 继续调整产品命名和 agent 结构，把 Builder/Coder 合并为 Agent 路线。这说明 TRAE SOLO 正在减少模式割裂，向统一 agent workflow 收敛。

## Product Signals

- TRAE changelog 在 2026-05-08 显示 Builder / Builder With MCP 合并为 Agent，SOLO Builder / SOLO Coder 合并为 SOLO Agent。
- TRAE command 文件支持更多目录嵌套，体现项目级规则/命令资产正在变重要。

## Product Interpretation

- **路线判断**：早期 AI coding 产品容易堆多个模式，但用户真正需要的是一个可理解的任务生命周期。把角色/模式背后的 harness 统一，有助于降低学习成本。
- **对 TRAE SOLO 的启示**：Work/Code 可以保留人机界面差异，但底层 agent state、context、action log、verification 应保持统一。

## LPME Implication

- LPME 应观察“同一任务从 Work 到 Code 的切换成本”：上下文是否丢失、计划是否继承、验证是否连续。
