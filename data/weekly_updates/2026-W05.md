# 2026-W05（2026-01-19～2026-01-25）

## Weekly Summary

本周多产品围绕团队配置、subagents、skills、gateway direct tools 和 TTS/channel 继续推进。主题是：agent 产品开始从个人技巧转为团队资产。

## Head Product Signals

- **OpenAI Codex**：1 月 22 日 custom prompts deprecated，1 月 23 日 Team Config for shared configuration；这是从个人 prompt 迁移到团队配置的明确信号。
- **Claude Code**：2.1.14-2.1.19 小步迭代；重点仍是 CLI/session 的可靠性积累。
- **Cursor**：1 月 22 日发布 Subagents, Skills, and Image Generation，说明 Cursor 把复杂任务拆解和多模态产物纳入主线。
- **GitHub Copilot**：1 月 21 日 Copilot CLI 支持 plan before you build / steer as you go，体现从问答向可控执行过程升级。
- **Windsurf / Devin Desktop**：无重大公开更新；但 Devin 这类云端 agent 的长期价值依然是可异步接任务。
- **OpenClaw**：1 月 20-24 日出现 Control UI copy-as-markdown、typed workflows with approval gates、/tools/invoke HTTP endpoint、TTS、LINE plugin、gateway hardening。
- **Kimi Code**：回溯补录：本周未发现 Kimi Code 同级重大公开更新；在 skills、memory、channels、cloud agent 逐步成形的阶段，Kimi 后续要重点观察“模型升级 + agent harness”是否同步推进。
- **Zhipu GLM Coding Plan / CodeGeeX**：回溯补录：本周未发现智谱/CodeGeeX 同级重大公开更新；CodeGeeX 的观察重点从代码补全/解释扩展到 review、test、MCP 和团队工作流。
- **TRAE SOLO**：TRAE 在 1 月下旬继续围绕 SOLO Skills / agent 工作流完善；重点应从“能创建技能”转向“团队能复用技能”。

## Competitive Reading

- **路线判断**：团队配置和 skills 是 agent 产品组织化的入口。谁能把个人经验变成可版本化资产，谁就更容易进入企业场景。
- **模型能力变化**：模型侧没有单点大升级，但产品正在为频繁换模型做抽象层：Team Config、BYOK、provider registry、skills。
- **产品能力补位**：产品能力要补的是资产管理：版本、作者、适用范围、最近调用、成功率、失败样例、权限边界。

## TRAE SOLO Implication

TRAE SOLO 可以把 Skill 做成轻量 workflow card，直接告诉 PM/运营/投研用户“输入什么、产出什么、怎么验收”。

## LPME Implication

LPME 增加“团队资产化”指标：同一个任务能否从一次成功沉淀为团队模板，而不是下次重新聊一遍。
