## PM Notes

- **产品官短评**：今天没有新的底层模型跃迁，但产品层竞争在加速。OpenAI、GitHub、Cursor、Claude Code、Devin Desktop 都在把 agent 从“生成代码”包装成“执行任务”；国内模型厂商路线则需要持续观察其能否从模型/插件入口升级为完整工作流。
- **用户 workflow 改变**：用户会越来越少接受“给我一段代码”的 assistant，而会期待一个能拿 issue / spec / page / repo 开始工作、能自己跑验证、能展示进度、能被中途接管、能把结果发到 PR/preview/doc/channel 的 agent workbench。
- **模型能力变化**：今日未发现需要记录的新模型大版本或明确能力跃迁。模型仍会在长任务规划、跨文件一致性、浏览器/桌面操作、外部工具可靠性上出错，因此产品能力必须补偿这些缺口。
- **产品能力应补偿的模型缺口**：TRAE SOLO 需要强化任务状态机、证据链、检查点/回滚、权限确认、环境诊断、可视化预览、自动验证和人工接管。模型不稳定时，产品层要让用户知道 agent 在做什么、为什么这么做、哪里失败、如何恢复。
- **TRAE SOLO 应学习什么**：不要只学习 IDE agent。GitHub 的 repo-native workflow、Devin Desktop 的 Agent Command Center、Cursor 的 run mode/review 控制层、Claude Code 的透明可观察性、Codex 的 artifact/Sites 延伸、OpenClaw 的 channel/runtime，以及国内模型厂商的模型套餐、插件分发与 agent harness 联动，都值得拆成独立能力。SOLO 的定位应是跨角色产品交付 workbench，而不是另一个聊天框。
- **LPME benchmark 是否调整**：需要在 v0.2 增加或细化任务：长任务恢复/回滚、多 agent 状态管理、云/本地交接、外部工具权限边界、可验证交付物、PR/review workflow、Feishu/多渠道推送、非工程角色可理解的证据与风险说明。
