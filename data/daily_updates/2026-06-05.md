# AI Coding Daily Update - 2026-06-05

## Summary

今日有多条产品级公开更新：Codex 继续从代码 agent 扩展到可托管交付物，Claude Code / Copilot / Kimi Code / OpenClaw 加强 agent 治理、长任务和工具链稳定性，Cursor 与 Devin Desktop 重点补齐团队协作和迁移体验。

## Product Updates

### OpenAI Codex

- Codex app 26.602 新增个人活动洞察与分享卡片；同时 Sites 进入预览，可在 Codex app 中创建、保存、部署并管理由 OpenAI 托管的网站、仪表盘、内部工具和小游戏。
- Codex 新增 Amazon Bedrock 模型提供方接入，并在移动端/桌面端补充默认跟进方式、Windows SSH 连接、终端位置等配置能力。

### Claude Code

- Claude Code 2.1.163 增加组织托管的最低/最高版本约束、`/plugin list`、Hook 继续回合的 additionalContext，以及 MCP / Skills / 后台任务恢复等可靠性修复。

### Cursor

- Cursor 3.7 发布 Canvas Design Mode 与 context usage report，允许用户直接标注 canvas UI 元素指导 agent 修改，并把上下文消耗拆解为可追问的交互报告。
- Cursor Enterprise 的 Organizations 能力 GA，支持一个企业下统一管理多个 team、group、预算、安全治理和模型/agent 权限。

### TRAE SOLO

- 无

### GitHub Copilot

- Copilot 在 6 月 4 日集中发布 agent tasks REST API、更大上下文窗口、可配置 reasoning level，以及 PR 中更丰富的 Copilot Chat 上下文。
- Copilot for failing Actions 扩展到 Pro / Pro+ / Max，继续把修复 CI 失败、远程 agent task 和代码审查串成端到端开发工作流。

### Windsurf / Devin Desktop

- Devin Desktop 3.0.21 新增 “Reset migration from Windsurf” 命令，用于重新导入 Windsurf 扩展和设置；6 月 2 日起 Windsurf 已正式更名为 Devin Desktop。

### OpenClaw

- OpenClaw 2026.6.2-beta.1 将插件/Skill 安装从危险代码扫描转向 operator install policy，并补齐 doctor、CLI、ClawHub、上传/市场安装等治理入口。
- 本次 beta 同时加强 Feishu / Telegram / Discord / WhatsApp 渠道投递、Control UI、Workboard、Skill Workshop、Gateway、provider、memory 和 CI 诊断的稳定性。

### Kimi Code

- Kimi Code 0.8.0 发布，新增后台自动升级、实验性 goal mode、压缩摘要保留 todo list 等长任务体验改进。
- Kimi K2.6 页面继续强化 coding、long-horizon execution、Agent Swarm、document-to-skill 和多交付物生成定位。

### Zhipu GLM Coding Plan / CodeGeeX

- GLM Coding Plan 继续明确团队版、GLM-5.1 / GLM-5-Turbo / GLM-4.7 / GLM-4.5-Air 可用模型、5 小时与周额度、专属 MCP 权益，并强调仅限官方支持工具环境使用。

## PM Notes

- **产品官短评**：今天的主线不是单点模型升级，而是 agent 产品把“可运行交付物、组织治理、长期任务恢复、权限策略、团队预算和多渠道交付”做成默认工作流。
- **对 TRAE SOLO 的启示**：SOLO 需要把 Work / Code 的跨角色定位继续落到可部署 artifact、任务治理、恢复机制和团队级权限/预算上，否则会被 Codex Sites、Cursor Canvas、Copilot agent task API 这类能力拉开交付闭环。
- **LPME 是否更新**：建议更新，新增“可部署交付物与回访管理”“组织策略/权限约束”“长任务恢复与后台升级”“多渠道交付可靠性”四类评测点。
