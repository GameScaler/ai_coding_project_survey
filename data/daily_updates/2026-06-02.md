# AI Coding Daily Update - 2026-06-02

## Summary

重大更新初筛：有重大更新。

今天是 AI coding 产品从“代码 agent”走向“agent command center / product delivery”的强信号日：OpenAI Codex 推 Sites preview，GitHub Copilot App 扩大预览并集中发布 agent 能力，Windsurf 升级为 Devin Desktop，OpenClaw 则从 agent gateway / runtime 侧补上多渠道、多插件、多 provider 的另一条路线。

## Product Updates

### OpenAI Codex

- Codex changelog 出现 Sites preview 相关更新，方向是用 Codex 创建、部署、管理网站和内部工具。
- 这意味着 Codex 的产物边界从本地代码/PR 往可访问、可分享、可托管的业务 artifact 延展。

### GitHub Copilot

- GitHub Copilot App 扩大 technical preview 可用范围。
- GitHub 同日公开多项 Copilot 更新，包括 Copilot SDK GA、Copilot CLI 体验增强、cloud/local sandboxes public preview、code review 团队规则、Azure Repos 技术预览、agent apps、Gemini models 和 MAI-Code-1-Flash 等。

### Windsurf / Devin Desktop

- Cognition 宣布 Windsurf is now Devin Desktop，强调在 IDE 基础上引入管理所有 agents 的 command center。
- Devin CLI 2026.5.26-3 可通过 Homebrew 安装，后续实测可与 Windsurf/Devin Desktop 路线合并。

### OpenClaw

- OpenClaw 2026.6.2 changelog 集中出现 Skill Workshop、Workboard、channels/mobile delivery、providers/plugins、Codex/Copilot integration 等信号。
- 它更像 agent operating system / gateway，而不是纯 IDE；关注重点是 agent 如何跨渠道运行、恢复、治理和被团队复用。

### Claude Code

- Claude Code 2.1.161 继续围绕 Opus 4.8、dynamic workflow/ultracode、auto mode、第三方 provider、IDE 稳定性做修复。

### Kimi Code

- 无同等级重大公开更新。观察价值在于国内模型厂商是否会用 K2.6 for coding + CLI/IDE/ACP 切入 agent command center。

### Zhipu GLM Coding Plan / CodeGeeX

- 无同等级重大公开更新。观察价值在于 GLM Coding Plan / CodeGeeX 是否会从模型套餐和插件入口升级为任务工作台。

## PM Notes

- **产品官短评**：今天的共同主题是“coding agent 的界面变成工作台”。OpenAI 用 Sites 把产物发布前移，GitHub 用 Copilot App 和 sandboxes 把执行环境产品化，Devin Desktop 把多 agent 管理变成主界面；OpenClaw 则提醒我们，工作台之外还会有 gateway、channel、plugin、skill governance 这些组织级运行时能力。
- **对 TRAE SOLO 的启示**：TRAE SOLO 不应只和 Cursor 比 IDE，而要把 Work/Code、浏览器、文档/表格/PPT、移动端、飞书协作组合成 product workbench。用户最终要的不是代码，而是可预览、可验证、可分享、可协作的交付物。
- **LPME 是否更新**：建议新增“从任务到可分享 artifact”的测试项，记录部署/分享/协作链路是否完整。
- **国内模型厂商观察**：Codex Sites、Copilot App、Devin Desktop、OpenClaw 共同说明主战场是“可管理的 agent workbench”；Kimi Code 与 Zhipu GLM Coding Plan / CodeGeeX 要纳入这个维度，而不是只看模型参数或插件能力。
