## PM Notes

- **产品官短评**：Auto-review 是 Cursor 对“长任务 agent 可控性”的回答。当前模型还不能完全信任，所以产品需要在每一步人工确认和完全放手之间增加中间态：自动审查、规则约束、风险分级、失败回退。
- **对 TRAE SOLO 的启示**：面向非工程师时，approval 不能只是命令确认。TRAE SOLO 应该把确认对象转为业务语义：是否允许访问某份资料、是否采用这个方案、是否发布这个页面、是否把摘要推送到群。
- **LPME 是否更新**：建议加入“长任务 approval friction”记录：确认次数、被打断次数、误确认风险、用户是否理解确认含义。
- **国内模型厂商观察**：Cursor 的 Auto-review 说明产品层必须补模型短板；Kimi Code 与 Zhipu GLM Coding Plan / CodeGeeX 不能只看模型或插件分发，也要看是否提供 review、rollback、权限和任务状态。
