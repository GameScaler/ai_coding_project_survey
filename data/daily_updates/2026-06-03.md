# AI Coding Daily Update - 2026-06-03

## Summary

重大更新初筛：有重大更新，但核心仍是 2026-06-02 大发布的延续消化。

今日要把 OpenClaw 正式纳入主要产品池。它不是 Cursor/Claude Code 这类纯 coding surface，而是 multi-channel gateway for AI agents：更接近 agent runtime / agent OS。对 TRAE SOLO 来说，这条线提醒我们，AI coding 产品能力不是“模型能力单点爆发”，而是模型能力 + 产品能力共同决定真实交付；当模型还不完全稳定时，产品层要补 context、workflow、permission、verification、channel delivery、recovery 和 governance。

## Product Updates

### OpenClaw

- OpenClaw 2026.6.2 changelog 集中出现 Skill Workshop、Workboard、channels/mobile delivery、providers/plugins、Codex/Copilot integration、gateway runtime recovery 等信号。
- 它的核心观察价值不是“又一个 IDE”，而是 agent 如何进入 Slack、Discord、Teams、Feishu、移动端、Cron、外部插件和多模型 provider，并在长任务中保持状态、恢复和权限边界。

### OpenAI Codex

- Codex 2026-06-02 的 Sites preview 与 2026-06-01 的 Bedrock provider / CLI 0.136.0 仍是本周最重要信号。
- Codex 的路线正在从 CLI/IDE/App 扩到 hosted artifact：网站、dashboard、内部工具、web app 不再只是代码，而是可被部署、检查和管理的产物。

### GitHub Copilot

- GitHub 2026-06-02 的集中更新仍在发酵：Copilot app、Copilot SDK GA、cloud/local sandboxes、agent apps、code review team skills、Copilot CLI、model updates、memory 与 automation 共同指向 GitHub-native agent platform。
- GitHub 的优势不是单个模型，而是 issue、PR、Actions、review、billing、enterprise policy 的 workflow ownership。

### Windsurf / Devin Desktop

- Windsurf is now Devin Desktop 继续强化 Agent Command Center：local IDE + cloud Devin + ACP-compatible agents + Spaces。
- 这条路线和 Cursor 的 Agents Window、GitHub Copilot app、Codex app 形成共识：长任务、多 agent、云端执行需要任务板和状态管理，而不是只靠聊天窗口。

### Cursor

- Cursor 3.6 Auto-review Run Mode 仍是近期关键变化：让 agent 能跑更久、减少 approval prompts，同时用 sandbox 和 classifier subagent 维持安全。
- Cursor 的产品判断很清晰：模型能力提升后，产品最先要补的是“控制层”，即 run mode、approval、review、worktree、context usage、Bugbot。

### Claude Code

- Claude Code 2.1.160 / 2.1.161 继续围绕 shell/config write 安全提示、acceptEdits safeguards、OTEL labels、parallel tool calls、fan-out progress、subagent/background session 稳定性迭代。
- Claude Code 的强项是透明、可组合、可下钻；对专业工程师有强吸引力，但跨角色使用仍需要更产品化的状态和产物解释层。

### TRAE SOLO

- TRAE SOLO 近期公开信号包括内置浏览器选中元素加入对话/评论、Work/Code、移动端、语音、文档/表格/PPT/Python 上下文。
- 这些能力应继续服务“产品交付”而不是“更像 IDE”：非工程师要看到任务计划、证据、产物预览、风险、反馈入口和可分享交付物。

### Others

- Kimi Code、Zhipu GLM Coding Plan / CodeGeeX 未观察到同等级别重大公开更新；继续分别观察“模型升级 + agent harness”和“模型套餐 + IDE 插件生态”能否升级为端到端 agent workflow。

## PM Notes

- **产品官短评**：OpenClaw 加入主要产品池后，竞品图谱更完整了。Cursor / Claude Code / Codex / Copilot / Devin Desktop 更多在 coding surface 和 software workflow 内演化，OpenClaw 则把 agent 运行时、渠道、插件、技能治理和恢复机制推到前台。
- **对 TRAE SOLO 的启示**：TRAE SOLO 要坚持“模型能力 + 产品能力”的基本认知。模型版本迭代决定能力上限，但产品层决定真实可用性。尤其面向 PM、运营、数据、投研等非工程角色时，产品需要把不稳定模型包进可审查、可恢复、可协作、可验证的工作流。
- **LPME 是否更新**：需要。v0.2 应增加“多渠道交付与订阅推送”“团队技能治理”“长任务恢复/回滚”“外部工具权限边界”四类指标，用来测试产品层是否真的补足模型短板。
- **国内模型厂商观察**：Kimi Code 与 Zhipu GLM Coding Plan / CodeGeeX 进入核心观察口径后，国内竞品图谱更完整：TRAE SOLO 做工作台，Kimi 做模型 + agent harness，智谱/CodeGeeX 做模型套餐 + 插件生态。TRAE 的壁垒必须是端到端交付，而不是单点模型或 IDE 能力。
