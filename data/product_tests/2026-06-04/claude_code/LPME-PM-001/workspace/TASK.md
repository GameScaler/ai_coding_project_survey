# LPME-PM-001: PRD to Clickable Prototype

You are evaluating an AI coding product.

Use `feature_brief.md`, `wireframe.md`, and `feedback.csv` to create:

1. `prototype/index.html`: a responsive clickable prototype.
2. `PRD.md`: refined PRD with scope, non-goals, metrics, edge cases, and assumptions.
3. `ACCEPTANCE_TESTS.md`: acceptance tests mapped to user stories.

## Requirements

- The prototype must work by opening `prototype/index.html` locally.
- It must include desktop and mobile responsive behavior.
- It must show at least two task states: "draft" and "ready for review".
- It must not invent integrations that are not in the brief.
- It should be understandable to a non-engineering PM.
